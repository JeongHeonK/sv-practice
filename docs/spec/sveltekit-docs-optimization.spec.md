# SvelteKit 문서 전체 최적화 Spec

## 개요

- **목적**: 교육용 SvelteKit 문서 8개를 개념 이해 중심으로 최적화
- **대상 독자**: React/Next.js 경험자, SvelteKit 실무 전환 예정자
- **핵심 원칙**: 구현 코드는 AI에게 맡기고, 문서는 "왜 이렇게 동작하나"에 집중
- **시각화 방식**: ASCII 다이어그램 위주
- **React 비교**: 각 파일 말미에 통합 테이블 1개, 섹션 내 비교 주석 제거

### 비기능 요구사항

- 각 개념당 핵심 코드 예제 **최대 1개** 유지
- 실습(진도 나가기식) 코드 섹션 **완전 제거**
- ASCII 다이어그램으로 흐름 시각화 (Mermaid 미사용)
- React 비교는 **파일 맨 끝 통합 테이블만** (섹션 내 인라인 비교 제거)

**말미 추가**:

**추가**:

  ```
  유형          | 패턴         | 예시 URL          | params
  -------------|-------------|-----------------|--------
  동적          | [id]        | /post/123       | { id: "123" }
  나머지(rest)  | [...rest]   | /a/b/c          | { rest: ["a","b","c"] }
  ```

**제거**:

- 라우트 매칭 우선순위 장황한 예시 코드

**말미 추가**:

#### `06-advanced-hooks.md` (370 → ~420줄)

**추가**:
1. **훅 실행 순서 ASCII 다이어그램** (섹션 0 또는 인트로에):
   ```
   HTTP 요청
      │
      ▼
   handle()          ← 모든 요청의 입구
      │
      ├─ handleFetch() ← load 내 fetch 호출 시만
      │
      ├─ reroute()    ← URL 매칭 전 재라우팅
      │
      └─ handleError() ← 에러 발생 시
   ```

2. **React/Next.js 비교 테이블** (파일 말미):
   | SvelteKit | React/Next.js | 차이점 |
   |-----------|--------------|-------|
   | `handle` | `middleware.ts` | ... |
   | `reroute` | `next.config.js rewrites` | ... |
   | `handleError` + `+error.svelte` | `error.tsx` (Error Boundary) | ... |
   | `handleFetch` | 없음 | 서버 fetch 가로채기 |

- `01`, `02`, `03`, `04`, `05`, `06` 최적화

### 제외 (Out of Scope)

## 엣지케이스 & 주의사항

- **07 스크롤 섹션 이동 후 총 줄 수 확인**: 613 + 스크롤 섹션 ≈ 650줄 초과 가능 → 이동 후 07의 다른 섹션 경량화 고려
- **React 비교 통합 테이블**: 기존 각 파일에 이미 있는 경우 내용 정비(항목 추가/제거)이므로 신규 추가가 아닐 수 있음
- **01의 $env 링크**: 08 파일명 `08-env-server-modules.md` 정확히 사용
